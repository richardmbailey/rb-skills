# Retired Skills

This directory preserves skills that are no longer part of the active RB skill pack.

Retired skills remain available for historical reference, but normal repository sync, metadata validation, and routing evaluation only discover skill folders that are direct children of the repository root. Do not invoke or install a retired skill as part of the normal workflow.

To reactivate a skill, move its directory back to the repository root, restore its active documentation and routing cases, run the repository validators, and sync the skill to the relevant agent installation.

## Skills

- `rb-create-issues`: retired because its decomposition and verification responsibilities overlap with `rb-create-implementation-plan` followed by `rb-execute-plan`.
